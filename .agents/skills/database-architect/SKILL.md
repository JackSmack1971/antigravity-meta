# Database Architect

## Overview

Design scalable, performant, and maintainable data layers from the ground up. This skill guides the selection of database technologies, schema modeling, and migration planning for complex systems.

## Core Workflow

### 1. Requirements & Access Patterns
Capture data domain constraints, read/write ratios, and scale targets.

### 2. Technology Selection
Choose the optimal database model (Relational, Document, Graph) based on architectural trade-offs.
> See [Tech Selection](file:///c:/workspaces/antigravity-meta/.agents/skills/database-architect/resources/technology-selection.md).

### 3. Schema & Index Design
Design physical models, partitioning strategies, and high-performance indexes.
> See [Data Modeling](file:///c:/workspaces/antigravity-meta/.agents/skills/database-architect/resources/data-modeling-and-indexing.md).

### 4. Scaling & Migration Plan
Design for horizontal growth and plan safe data migrations.
> See [Scalability](file:///c:/workspaces/antigravity-meta/.agents/skills/database-architect/resources/scalability-and-migration.md).
