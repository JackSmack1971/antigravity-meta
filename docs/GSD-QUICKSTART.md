# Get Shit Done (GSD) Workflow Quickstart

The GSD protocol is an atomic, linear workflow designed to eliminate "vibe coding."

## The 5-Phase Pipeline
1. **Spec**: Lock down exactly what is to be built.
2. **Plan**: Author an atomic implementation plan.
3. **Execute**: Develop using the `Awesome-Skills` repository.
4. **Verify**: Ensure code compilations run and all unit tests pass.
5. **Commit**: Save atomic changes to Git using conventional standards.

## Usage
*   Trigger a GSD cycle by typing `/gsd-plan [your feature requirement]` in the Agent Manager.
*   The agent will generate or update the `task.md` document at the root of the project to chart immediate milestones.
*   Never skip the Verification phase!
