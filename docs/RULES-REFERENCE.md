# Governance Identity Rules

The `.agents/rules/` directory houses the mandatory constraints injected into all agent contexts.

1. **`hybrid-protocol.md`**: Master orchestrator rule dictating when to use GSD vs BMAD.
2. **`gsd-core-rules.md`**: Implements the planning lock, ensuring empirical verification before any commit sequence.
3. **`rugged-software-constitution.md`**: Hardened production-grade constraints for abstractions and state management.
4. **`security-mandate.md`**: Zero-trust execution boundaries; binds the agent to explicit directory scopes.
5. **`strong-reasoner-planner.md`**: System 2 thinking enforcer; mandates discrete rationalization sequences before API usage.
6. **`code-quality-git-testing.md`**: Ensures adherence to conventional commits, automated linting, and TDD (Test-Driven Development).
